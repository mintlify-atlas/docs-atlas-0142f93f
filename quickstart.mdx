---

The Okta Go Management SDK lets you interact with the Okta Management API from server-side Go applications. This guide walks you through installing the SDK, creating a client, and making your first API call.

<Note>
  You need an Okta organization and an API token to follow this guide. Sign up for a free [developer organization](https://developer.okta.com/signup) if you don't have one. Then follow the [API token guide](https://developer.okta.com/docs/api/getting_started/getting_a_token) to create a token.
</Note>

<Steps>
  <Step title="Install the SDK">
    The SDK requires Go 1.24 or later and uses Go Modules. If your project doesn't have a `go.mod` file yet, initialize one first:

    ```bash
    go mod init your-module-name
    ```

    Then add the SDK:

    ```bash
    go get github.com/okta/okta-sdk-golang/v6@latest
    ```

    This adds the SDK to your `go.mod` and downloads the package.
  </Step>

  <Step title="Import the package">
    Add the SDK import to your Go source file:

    ```go
    import (
        "context"
        "fmt"
        "github.com/okta/okta-sdk-golang/v6/okta"
    )
    ```
  </Step>

  <Step title="Create a configuration and client">
    Construct a `Configuration` using `okta.NewConfiguration`, then pass it to `okta.NewAPIClient`:

    ```go
    config, err := okta.NewConfiguration(
        okta.WithOrgUrl("https://{yourOktaDomain}"),
        okta.WithToken("{apiToken}"),
    )
    if err != nil {
        fmt.Printf("Error: %v\n", err)
    }
    client := okta.NewAPIClient(config)
    ```

    Replace `{yourOktaDomain}` with your Okta domain (for example, `dev-123456.okta.com`) and `{apiToken}` with the token you created.

    <Tip>
      Hard-coding credentials works for quick tests, but for real projects store them in environment variables or a YAML config file. See the [configuration guide](/configuration) for all options.
    </Tip>
  </Step>

  <Step title="Make your first API call">
    Once you have a client, call any API through the corresponding service field. The following examples show two common first calls.

    <CodeGroup>

    ```go Get a user
    import (
        "context"
        "fmt"
        "github.com/okta/okta-sdk-golang/v6/okta"
    )

    func main() {
        config, err := okta.NewConfiguration(
            okta.WithOrgUrl("https://{yourOktaDomain}"),
            okta.WithToken("{apiToken}"),
        )
        if err != nil {
            fmt.Printf("Error: %v\n", err)
        }
        client := okta.NewAPIClient(config)

        user, resp, err := client.UserAPI.GetUser(context.Background(), "{UserId|Username|Email}").Execute()
        if err != nil {
            fmt.Printf("Error Getting User: %v\n", err)
        }
        fmt.Printf("User: %+v\n Response: %+v\n\n", user, resp)
    }
    ```

    ```go List all users
    import (
        "context"
        "fmt"
        "github.com/okta/okta-sdk-golang/v6/okta"
    )

    func main() {
        config, err := okta.NewConfiguration(
            okta.WithOrgUrl("https://{yourOktaDomain}"),
            okta.WithToken("{apiToken}"),
        )
        if err != nil {
            fmt.Printf("Error: %v\n", err)
        }
        client := okta.NewAPIClient(config)

        users, resp, err := client.UserAPI.ListUsers(context.Background()).Execute()
        if err != nil {
            fmt.Printf("Error Getting Users: %v\n", err)
        }
        fmt.Printf("Users: %+v\n Response: %+v\n\n", users, resp)
        for index, user := range users {
            fmt.Printf("User %d: %+v\n", index, user)
        }
    }
    ```

    </CodeGroup>

    API methods are organized by endpoint. Users live under `client.UserAPI`, groups under `client.GroupAPI`, applications under `client.ApplicationAPI`, and so on. Every method accepts `context.Context` as its first argument — use `context.Background()` if you don't have a specific context, or `context.TODO()` as a placeholder.
  </Step>
</Steps>
