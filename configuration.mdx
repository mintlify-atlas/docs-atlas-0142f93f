---
title: "Configure the Okta Go SDK client"
description: "Set organization URL, API tokens, OAuth credentials, caching, timeouts, and proxy settings via YAML, environment variables, or Go code."
---

The SDK resolves configuration from four sources in a defined order of precedence. Later sources override earlier ones, so values set in code always win over environment variables, which override anything in a YAML file. This lets you set sensible defaults in a config file and override specific values at runtime.

## Configuration precedence

The SDK loads configuration in this order (highest number wins):

1. `~/.okta/okta.yaml` â€” user-level YAML file in the current user's home directory
2. `.okta.yaml` â€” project-level YAML file in the application root directory
3. Environment variables â€” prefixed with `OKTA_CLIENT_`
4. Constructor options â€” `WithXxx()` setters passed to `okta.NewConfiguration`

## Configuration methods

<Tabs>
  <Tab title="YAML">
    Place a YAML file at `~/.okta/okta.yaml` (user-level) or `.okta.yaml` in your project root (project-level). The SDK reads both files on startup.

    **API token authentication**

    ```yaml
    okta:
      client:
        connectionTimeout: 30 # seconds
        orgUrl: "https://{yourOktaDomain}"
        proxy:
          port: null
          host: null
          username: null
          password: null
        token: {apiToken}
    ```

    **OAuth 2.0 with a private key**

    ```yaml
    okta:
      client:
        connectionTimeout: 30 # seconds
        orgUrl: "https://{yourOktaDomain}"
        proxy:
          port: null
          host: null
          username: null
          password: null
        authorizationMode: "PrivateKey"
        clientId: "{yourClientId}"
        scopes:
          - scope.1
          - scope.2
        privateKey: |
            -----BEGIN RSA PRIVATE KEY-----
            MIIEogIBAAKCAQEAl4F5CrP6Wu2kKwH1Z+CNBdo0iteHhVRIXeHdeoqIB1iXvuv4
            THQdM5PIlot6XmeV1KUKuzw2ewDeb5zcasA4QHPcSVh2+KzbttPQ+RUXCUAr5t+r
            0r6gBc5Dy1IPjCFsqsPJXFwqe3RzUb...
            -----END RSA PRIVATE KEY-----
        privateKeyId: "{JWK key id (kid)}" # needed if the service app has more than one JWK registered
        requestTimeout: 0 # seconds
        rateLimit:
          maxRetries: 4
    ```

    <Note>
      When using the YAML file for OAuth 2.0, the `privateKey` field accepts either a PEM block (as shown above) or a file path to a `.pem` file.
    </Note>

    Example shell setup for API token auth:

    ```bash
    export OKTA_CLIENT_ORGURL="https://dev-123456.okta.com"
    export OKTA_CLIENT_TOKEN="your-api-token"
    ```

    With environment variables set, you can create a client without passing any options:

    ```go
    config, err := okta.NewConfiguration()
    if err != nil {
        fmt.Printf("Error: %v\n", err)
    }
    client := okta.NewAPIClient(config)
    ```

  <Tab title="Go code">
    Pass `WithXxx()` setter functions directly to `okta.NewConfiguration`. This is the most explicit approach and takes the highest precedence over all other sources.

    **API token**

    ```go
    config, err := okta.NewConfiguration(
        okta.WithOrgUrl("https://{yourOktaDomain}"),
        okta.WithToken("{apiToken}"),
    )
    if err != nil {
        fmt.Printf("Error: %v\n", err)
    }
    client := okta.NewAPIClient(config)
    ```

    **OAuth 2.0 with a private key**

    ```go
    config, err := okta.NewConfiguration(
        okta.WithOrgUrl("https://{yourOktaDomain}"),
        okta.WithAuthorizationMode("PrivateKey"),
        okta.WithClientId("{{clientId}}"),
        okta.WithScopes([]string{"okta.users.manage"}),
        okta.WithPrivateKey({{PEM PRIVATE KEY BLOCK}}),
        okta.WithPrivateKeyId("{{private key id}}"),
    )
    if err != nil {
        fmt.Printf("Error: %v\n", err)
    }
    client := okta.NewAPIClient(config)
    ```

    **Timeouts and rate limiting**

    ```go
    config, err := okta.NewConfiguration(
        okta.WithOrgUrl("https://{yourOktaDomain}"),
        okta.WithToken("{apiToken}"),
        okta.WithRequestTimeout(45),
        okta.WithRateLimitMaxRetries(3),
    )
    if err != nil {
        fmt.Printf("Error: %v\n", err)
    }
    client := okta.NewAPIClient(config)
    ```

## All configuration setters

The following `WithXxx()` functions can be passed to `okta.NewConfiguration`:

| Function | Description |
|---|---|
| `WithOrgUrl(url string)` | Okta organization URL |
| `WithToken(token string)` | Okta API token |
| `WithAuthorizationMode(authzMode string)` | Auth mode: `SSWS`, `PrivateKey`, `JWT`, or `Bearer` |
| `WithClientId(clientId string)` | OAuth 2.0 client ID |
| `WithScopes(scopes []string)` | OAuth 2.0 app scopes |
| `WithPrivateKey(privateKey string)` | PEM private key value or path to a `.pem` file |
| `WithPrivateKeyId(privateKeyId string)` | Private key ID (`kid`) |
| `WithPrivateKeySigner(signer jose.Signer)` | Custom private key signer implementing `jose.Signer` |
| `WithConnectionTimeout(i int64)` | HTTP connection timeout in seconds |
| `WithRequestTimeout(requestTimeout int64)` | HTTP request timeout in seconds |
| `WithRateLimitMaxRetries(maxRetries int32)` | Max retries on 429 responses |
| `WithRateLimitMaxBackOff(maxBackoff int64)` | Max backoff time in seconds |
| `WithRateLimitPrevent(enable bool)` | Enable proactive rate limit prevention |
| `WithCache(cache bool)` | Enable request memory cache |
| `WithCacheManager(cacheManager cache.Cache)` | Custom cache implementing the `cache.Cache` interface |
| `WithCacheTtl(i int32)` | Cache time to live in seconds |
| `WithCacheTti(i int32)` | Cache clean up interval in seconds |
| `WithProxyHost(host string)` | HTTP proxy host |
| `WithProxyPort(i int32)` | HTTP proxy port |
| `WithProxyUsername(username string)` | HTTP proxy username |
| `WithProxyPassword(pass string)` | HTTP proxy password |
| `WithHttpClientPtr(httpClient *http.Client)` | Custom `net/http` client |
| `WithUserAgentExtra(userAgent string)` | Additional string appended to the HTTP User-Agent header |
| `WithTestingDisableHttpsCheck(httpsCheck bool)` | Disable SSL certificate checks (testing only) |

## Default values

When no configuration is provided for a setting, the SDK starts with these defaults:

| Setting | Default |
|---|---|
| `WithConnectionTimeout` | `60` seconds |
| `WithCache` | `true` |
| `WithCacheTtl` | `300` seconds |
| `WithCacheTti` | `300` seconds |
| `WithUserAgentExtra` | `""` (empty) |
| `WithTestingDisableHttpsCheck` | `false` |
| `WithRequestTimeout` | `0` (no timeout) |
| `WithRateLimitMaxBackOff` | `30` seconds |
| `WithRateLimitMaxRetries` | `2` |
| `WithAuthorizationMode` | `"SSWS"` |

<Warning>
  Setting `WithRequestTimeout` to `0` disables the request timeout entirely. In production, set an explicit timeout to prevent requests from hanging indefinitely.
</Warning>

## Rate limiting and retries

By default, the SDK retries requests that receive a 429 (Too Many Requests) response. The retry behavior is controlled by two settings:

- `WithRequestTimeout` â€” the maximum total time to spend retrying (in seconds). Set to `0` to retry indefinitely until `maxRetries` is exhausted.
- `WithRateLimitMaxRetries` â€” the maximum number of retry attempts. Set to `0` for unlimited retries within the request timeout.

If both values are non-zero, the SDK stops retrying when either condition is met first. To disable retries entirely, set both to `0`.

The SDK uses the server's `Date` and `X-Rate-Limit-Reset` headers to calculate how long to wait before retrying:

```go
backoff_seconds = header['X-Rate-Limit-Reset'] - header['Date'] + 1s
```

## Custom HTTP client and cache

You can inject a custom `http.Client` or cache implementation when calling `okta.NewConfiguration`:

```go
myClient := &http.Client{}

myCache := NewCustomCacheDriver()

config, err := okta.NewConfiguration(
    okta.WithOrgUrl("https://{yourOktaDomain}"),
    okta.WithToken("{apiToken}"),
    okta.WithHttpClientPtr(myClient),
    okta.WithCacheManager(myCache),
)
if err != nil {
    fmt.Printf("Error: %v\n", err)
}
client := okta.NewAPIClient(config)
```

A custom cache must implement the `cache.Cache` interface:

```go
type CustomCacheDriver struct{}

func NewCustomCacheDriver() Cache {
    return CustomCacheDriver{}
}

func (c CustomCacheDriver) Get(key string) *http.Response       { return nil }
func (c CustomCacheDriver) Set(key string, value *http.Response) {}
func (c CustomCacheDriver) GetString(key string) string          { return "" }
func (c CustomCacheDriver) SetString(key string, value string)   {}
func (c CustomCacheDriver) Delete(key string)                    {}
func (c CustomCacheDriver) Clear()                               {}
func (c CustomCacheDriver) Has(key string) bool                  { return false }
```
