title: "PolicyAPI — Okta policy and rule management"
description: "Reference for client.PolicyAPI: list, create, and manage Okta policies (sign-on, password, enrollment) and their rules using the Go SDK."
---

The `PolicyAPI` provides full lifecycle management for Okta policies and their associated rules. Policies control authentication, password requirements, MFA enrollment, and profile enrollment behavior across your Okta org. Each policy contains one or more rules that define the specific conditions and actions that apply to a matched user or session.

## Policy types

Okta supports the following policy types. You pass the type string when creating a policy or filtering with `ListPolicies`.

| Type | Description |
|------|-------------|
| `OKTA_SIGN_ON` | Controls sign-on behavior including session lifetime and authentication requirements |
| `PASSWORD` | Defines password complexity, lockout, and recovery rules |
| `MFA_ENROLL` | Controls which authenticators users must or may enroll |
| `PROFILE_ENROLLMENT` | Governs how users are enrolled and what profile attributes are collected |
| `ACCESS_POLICY` | Identity Engine policy controlling access to apps (Okta Identity Engine only) |
| `IDP_DISCOVERY` | Routes users to the correct identity provider during sign-in |
| `ENTITY_RISK` | Applies actions based on entity risk signals (Okta Identity Engine only) |

## Access the API

```go
client, err := okta.NewClient(ctx)
if err != nil {
    log.Fatal(err)
}

policyAPI := client.PolicyAPI
```

## ListPolicies

Lists all policies. Use the `Type_` query option to filter by policy type.

```go
func (a *PolicyAPIService) ListPolicies(ctx context.Context) ApiListPoliciesRequest
func (a *PolicyAPIService) ListPoliciesExecute(r ApiListPoliciesRequest) ([]ListPolicies200ResponseInner, *APIResponse, error)
```

### Query options

<ParamField path="query.Type_" type="string">
  Filters policies by type. One of `OKTA_SIGN_ON`, `PASSWORD`, `MFA_ENROLL`, `PROFILE_ENROLLMENT`, `ACCESS_POLICY`, `IDP_DISCOVERY`, `ENTITY_RISK`, `POST_AUTH_SESSION`, `DEVICE_SIGNAL_COLLECTION`, or `CLIENT_UPDATE`.
</ParamField>

<ParamField path="query.Status" type="string">
  Filters policies by status. Use `ACTIVE` or `INACTIVE`.
</ParamField>

<ParamField path="query.Q" type="string">
  Filters policies by name prefix (startsWith match).
</ParamField>

<ParamField path="query.Expand" type="string">
  Expands embedded objects in the response. Use `rules` to include policy rules inline.
</ParamField>

<ParamField path="query.Limit" type="string">
  Maximum number of policies to return. See Okta pagination documentation.
</ParamField>

<ParamField path="query.After" type="string">
  End-page cursor for paginating through results.
</ParamField>

### Returns

<ResponseField name="[]ListPolicies200ResponseInner" type="object[]">
  A list of policy objects. Each element is a discriminated union that resolves to the concrete policy type (for example, `OktaSignOnPolicy`, `PasswordPolicy`, `MultifactorEnrollmentPolicy`).
</ResponseField>

### Example: list all sign-on policies

```go
policies, resp, err := client.PolicyAPI.
    ListPolicies(ctx).
    Type_("OKTA_SIGN_ON").
    Execute()
if err != nil {
    log.Fatalf("ListPolicies error: %v", err)
}

for _, p := range policies {
    if sp := p.OktaSignOnPolicy; sp != nil {
        fmt.Printf("Policy: %s (id=%s, status=%s)\n",
            sp.GetName(), sp.GetId(), sp.GetStatus())
    }
}
_ = resp
```

## CreatePolicy

Creates a new policy. The request body must include `type` and `name`.

```go
func (a *PolicyAPIService) CreatePolicy(ctx context.Context) ApiCreatePolicyRequest
func (a *PolicyAPIService) CreatePolicyExecute(r ApiCreatePolicyRequest) (*CreatePolicyRequest, *APIResponse, error)
```

### Query options

<ParamField path="query.Activate" type="boolean" default="true">
  When `true`, activates the policy immediately after creation.
</ParamField>

### Request body options

<ParamField path="body.Policy" type="object" required>
  A policy object. Must include `name` (string) and `type` (string). Provide additional type-specific fields depending on the policy variant you are creating.
</ParamField>

### Returns

<ResponseField name="*CreatePolicyRequest" type="object">
  The created policy object.
</ResponseField>

### Example: create a password policy

```go
import "github.com/okta/okta-sdk-golang/v6/okta"

name := "Corporate Password Policy"
policyType := "PASSWORD"

policy := okta.ListPolicies200ResponseInner{
    PasswordPolicy: &okta.PasswordPolicy{
        Policy: okta.Policy{
            Name: name,
            Type: policyType,
        },
    },
}

created, resp, err := client.PolicyAPI.
    CreatePolicy(ctx).
    Policy(policy).
    Activate(true).
    Execute()
if err != nil {
    log.Fatalf("CreatePolicy error: %v", err)
}
fmt.Printf("Created policy id: %s\n", created.PasswordPolicy.GetId())
_ = resp
```

## Policy rule methods

Rules define the conditions and actions that apply within a policy. Each policy has a prioritized list of rules evaluated in order.

<Note>
  You cannot create additional rules for `PROFILE_ENROLLMENT`, `POST_AUTH_SESSION`, or `CLIENT_UPDATE` policies.
</Note>

## Policy model fields

The `Policy` base type contains the following fields. Concrete policy types embed `Policy` and add type-specific settings.

<ResponseField name="id" type="string">
  Unique identifier for the policy. Read-only; set by Okta on creation.
</ResponseField>

<ResponseField name="name" type="string" required>
  Human-readable name for the policy.
</ResponseField>

<ResponseField name="type" type="string" required>
  The policy type. See [Policy types](#policy-types) above.
</ResponseField>

<ResponseField name="status" type="string">
  `ACTIVE` or `INACTIVE`. Use the activate/deactivate endpoints to change status.
</ResponseField>

<ResponseField name="priority" type="integer">
  Evaluation order relative to other policies of the same type. Lower values are evaluated first.
</ResponseField>

<ResponseField name="system" type="boolean">
  `true` for Okta-managed system policies that cannot be deleted. Default is `false`.
</ResponseField>

<ResponseField name="created" type="string">
  ISO 8601 timestamp of when the policy was created.
</ResponseField>

<ResponseField name="lastUpdated" type="string">
  ISO 8601 timestamp of the most recent modification.
</ResponseField>

## PolicyRule model fields

<ResponseField name="id" type="string">
  Unique identifier for the rule.
</ResponseField>

<ResponseField name="name" type="string">
  Human-readable name for the rule.
</ResponseField>

<ResponseField name="type" type="string">
  The rule type, which is specific to the parent policy type.
</ResponseField>

<ResponseField name="priority" type="integer">
  Evaluation order within the policy. Rules with lower priority numbers are evaluated first.
</ResponseField>

<ResponseField name="status" type="string">
  `ACTIVE` or `INACTIVE`.
</ResponseField>

<ResponseField name="system" type="boolean">
  `true` for system-managed rules that cannot be deleted.
</ResponseField>
