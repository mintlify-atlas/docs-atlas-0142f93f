---
title: "Role assignment APIs for users, groups, and clients"
description: "Reference for role assignment in the Okta Go SDK: assign standard and custom roles to users, groups, and OAuth clients, and manage resource sets."
---

Okta provides several role-related APIs to manage administrative access. Standard roles (such as `SUPER_ADMIN` or `ORG_ADMIN`) grant broad administrative privileges and can be assigned directly to users or groups. Custom roles offer fine-grained control through a combination of a custom role definition, a resource set that scopes which resources the role applies to, and a binding that connects members (users or groups) to both. The SDK exposes these capabilities through four focused API interfaces.

## Access the APIs

```go
client, err := okta.NewClient(ctx)
if err != nil {
    log.Fatal(err)
}

userRoleAPI  := client.RoleAssignmentAUserAPI
groupRoleAPI := client.RoleAssignmentBGroupAPI
customRoleAPI    := client.RoleECustomAPI
resourceSetAPI   := client.RoleCResourceSetAPI
bindingAPI       := client.RoleDResourceSetBindingAPI
```

## RoleAssignmentAUserAPI — user role assignments

This API manages standard and custom role assignments for individual users.

### ListAssignedRolesForUser

Lists all roles assigned to a user.

```go
func (a *RoleAssignmentAUserAPIService) ListAssignedRolesForUser(ctx context.Context, userId string) ApiListAssignedRolesForUserRequest
func (a *RoleAssignmentAUserAPIService) ListAssignedRolesForUserExecute(r ApiListAssignedRolesForUserRequest) ([]ListGroupAssignedRoles200ResponseInner, *APIResponse, error)
```

<ParamField path="path.userId" type="string" required>
  ID of an existing Okta user.
</ParamField>

<ResponseField name="[]ListGroupAssignedRoles200ResponseInner" type="object[]">
  A list of role assignment objects. Each object represents a standard or custom role assignment.
</ResponseField>

### AssignRoleToUser

Assigns a standard role to a user. To assign a custom role, create a resource set binding instead using `RoleDResourceSetBindingAPI.CreateResourceSetBinding`.

```go
func (a *RoleAssignmentAUserAPIService) AssignRoleToUser(ctx context.Context, userId string) ApiAssignRoleToUserRequest
func (a *RoleAssignmentAUserAPIService) AssignRoleToUserExecute(r ApiAssignRoleToUserRequest) (*AssignRoleToUser201Response, *APIResponse, error)
```

<ParamField path="path.userId" type="string" required>
  ID of an existing Okta user.
</ParamField>

<ParamField path="body.AssignRoleRequest" type="object" required>
  The role assignment request body. For standard roles, set `type` to the role type string (for example, `ORG_ADMIN`). For custom roles, set `type` to `CUSTOM` and provide `role` and `resource-set` identifiers.
</ParamField>

<ParamField path="query.DisableNotifications" type="boolean">
  When `true`, grants the user third-party admin status.
</ParamField>

<ResponseField name="*AssignRoleToUser201Response" type="object">
  The created role assignment.
</ResponseField>

### Example: assign a standard role to a user

```go
req := okta.AssignRoleToUserRequest{
    Type: okta.PtrString("ORG_ADMIN"),
}

assignment, resp, err := client.RoleAssignmentAUserAPI.
    AssignRoleToUser(ctx, "00u1a2b3c4d5e6f7g8h9").
    AssignRoleRequest(req).
    Execute()
if err != nil {
    log.Fatalf("AssignRoleToUser error: %v", err)
}
fmt.Printf("Assigned role id: %s\n", assignment.IamRoleAssignmentType.GetId())
_ = resp
```

## RoleCResourceSetAPI — resource sets

A resource set is a collection of Okta resources (identified by ORN or REST URL) that scopes the administrative access granted by a custom role binding. Resource sets are required when assigning custom roles.

<Note>
  A resource set can contain a maximum of 1000 resources. Resources are identified by Okta Resource Names (ORN) or REST URL format.
</Note>

## RoleDResourceSetBindingAPI — resource set bindings

A binding connects a custom role, a resource set, and one or more members (users or groups). When you create a binding, the members gain the permissions defined in the custom role, scoped to the resources in the resource set.

<Warning>
  If a custom role has permissions that don't apply to the resources in the bound resource set, those permissions have no effect. Make sure the resource set contains appropriate resources for the permissions you are granting.
</Warning>

### CreateResourceSetBinding

Creates a binding for a resource set, custom role, and members.

```go
func (a *RoleDResourceSetBindingAPIService) CreateResourceSetBinding(ctx context.Context, resourceSetIdOrLabel string) ApiCreateResourceSetBindingRequest
func (a *RoleDResourceSetBindingAPIService) CreateResourceSetBindingExecute(r ApiCreateResourceSetBindingRequest) (*ResourceSetBindingEditResponse, *APIResponse, error)
```

<ParamField path="path.resourceSetIdOrLabel" type="string" required>
  The `id` or `label` of the resource set.
</ParamField>

<ParamField path="body.Instance" type="object" required>
  A `ResourceSetBindingCreateRequest` specifying the `role` (custom role ID or label) and `members` array (user or group resource URLs).
</ParamField>

<ResponseField name="*ResourceSetBindingEditResponse" type="object">
  The created binding including `_links` to the role, resource set, and members.
</ResponseField>

### Example: bind a custom role to a resource set

```go
binding := okta.ResourceSetBindingCreateRequest{
    Role:    okta.PtrString("helpdesk-admin"),
    Members: []string{
        "https://{yourOktaDomain}/api/v1/users/00u1abc",
        "https://{yourOktaDomain}/api/v1/groups/00g1xyz",
    },
}

result, resp, err := client.RoleDResourceSetBindingAPI.
    CreateResourceSetBinding(ctx, "my-resource-set").
    Instance(binding).
    Execute()
if err != nil {
    log.Fatalf("CreateResourceSetBinding error: %v", err)
}
_ = result
_ = resp
```
