The `AuthorizationServerAPI` and its companion interfaces (`AuthorizationServerScopesAPI`, `AuthorizationServerClaimsAPI`, `AuthorizationServerPoliciesAPI`, and `AuthorizationServerKeysAPI`) provide complete control over custom OAuth 2.0 authorization servers in Okta. Use them to create and configure authorization servers, define token scopes and claims, manage access policies, and rotate signing keys. All methods follow the builder pattern: call the method, chain any optional parameters, then call `.Execute()`.

## AuthorizationServerAPI

### `ListAuthorizationServers`

```go
func (a *AuthorizationServerAPIService) ListAuthorizationServers(ctx context.Context) ApiListAuthorizationServersRequest
```

Lists all custom authorization servers configured in the org.

<ParamField path="ctx" type="context.Context" required>
  Request context for deadlines, cancellation, and authentication.
</ParamField>

**Optional chained parameters:**

<ParamField path=".Q(q string)" type="string">
  Searches the `name` field for a case-insensitive match.
</ParamField>

<ParamField path=".Limit(limit int32)" type="int32">
  Maximum number of results per page.
</ParamField>

<ParamField path=".After(after string)" type="string">
  Cursor for the next page of results.
</ParamField>

<ResponseField name="[]AuthorizationServer" type="slice" required>
  A slice of authorization server objects.
</ResponseField>

<ResponseField name="*APIResponse" type="object" required>
  HTTP response metadata.
</ResponseField>

```go
import (
    "fmt"
    "context"
    "github.com/okta/okta-sdk-golang/v6/okta"
)

func main() {
    config, err := okta.NewConfiguration(
        okta.WithOrgUrl("https://{yourOktaDomain}"),
        okta.WithToken("{apiToken}"),
    )
    if err != nil {
        fmt.Printf("Error: %v\n", err)
    }
    client := okta.NewAPIClient(config)

    servers, resp, err := client.AuthorizationServerAPI.ListAuthorizationServers(context.Background()).Execute()
    if err != nil {
        fmt.Printf("Error listing authorization servers: %v\n", err)
    }
    fmt.Printf("Servers: %+v\n Response: %+v\n\n", servers, resp)
}
```

### `CreateAuthorizationServer`

```go
func (a *AuthorizationServerAPIService) CreateAuthorizationServer(ctx context.Context) ApiCreateAuthorizationServerRequest
```

Creates a new custom authorization server.

<ParamField path="ctx" type="context.Context" required>
  Request context.
</ParamField>

**Required chained parameters:**

<ParamField path=".AuthorizationServer(authorizationServer AuthorizationServer)" type="AuthorizationServer" required>
  The authorization server configuration. Must include `name`, `description`, and `audiences`.
</ParamField>

<ResponseField name="*AuthorizationServer" type="object" required>
  The created authorization server with its assigned `id` and `issuer`.
</ResponseField>

```go
server := okta.AuthorizationServer{}
server.SetName("My Custom Auth Server")
server.SetDescription("Authorization server for my API")
server.SetAudiences([]string{"api://myapi"})

created, resp, err := client.AuthorizationServerAPI.
    CreateAuthorizationServer(context.Background()).
    AuthorizationServer(server).
    Execute()
if err != nil {
    fmt.Printf("Error creating authorization server: %v\n", err)
}
fmt.Printf("Created: %+v\n Response: %+v\n\n", created, resp)
```

## AuthorizationServerKeysAPI

The `AuthorizationServerKeysAPI` manages the JSON Web Keys (JWKs) used to sign tokens. Access it via `client.AuthorizationServerKeysAPI`.

### `RotateAuthorizationServerKeys`

```go
func (a *AuthorizationServerKeysAPIService) RotateAuthorizationServerKeys(ctx context.Context, authServerId string) ApiRotateAuthorizationServerKeysRequest
```

Rotates the current signing keys for an authorization server. After rotation, the `ACTIVE` key becomes `EXPIRED`, the `NEXT` key becomes `ACTIVE`, and the server immediately starts signing tokens with the new active key.

<ParamField path="ctx" type="context.Context" required>
  Request context.
</ParamField>

<ParamField path="authServerId" type="string" required>
  The unique ID of the authorization server.
</ParamField>

**Required chained parameters:**

<ParamField path=".Use(use JwkUse)" type="JwkUse" required>
  The key use type. Pass `okta.JWKUSE_SIG` for signature keys.
</ParamField>

<ResponseField name="[]AuthorizationServerJsonWebKey" type="slice" required>
  The updated set of keys including the new active key.
</ResponseField>

<Note>
  Okta automatically rotates keys in `AUTO` mode. If you rotate keys manually, invalidate any intermediate token caches and re-fetch the JWKS endpoint so consuming services pick up the new keys immediately.
</Note>

```go
keys, resp, err := client.AuthorizationServerKeysAPI.
    RotateAuthorizationServerKeys(context.Background(), "{authServerId}").
    Use(okta.JWKUSE_SIG).
    Execute()
if err != nil {
    fmt.Printf("Error rotating keys: %v\n", err)
}
for _, key := range keys {
    fmt.Printf("Key: kid=%s status=%s\n", key.GetKid(), key.GetStatus())
}
fmt.Printf("Response: %+v\n", resp)
```
