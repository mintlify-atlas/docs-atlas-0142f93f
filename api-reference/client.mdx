title: "APIClient — Okta Go SDK client reference"
description: "Reference for okta.NewAPIClient() and the APIClient struct. Lists all available API service properties and how to initialize the client."
---

`APIClient` is the central struct of the Okta Go SDK. You construct one instance per application by passing a `*Configuration` to `NewAPIClient`, then call methods through its API service properties. In most applications there should be a single shared `APIClient`. All service fields are interfaces, which makes it straightforward to substitute test doubles in unit tests.

## Constants

```go
const VERSION = "6.1.6"
```

`VERSION` is the current SDK release string. It is embedded in the default `User-Agent` header sent with every request.

## Constructor

```go
func NewAPIClient(cfg *Configuration) *APIClient
```

`NewAPIClient` allocates and initializes an `APIClient` from the supplied `*Configuration`. You must call `NewConfiguration` (or build a `Configuration` manually) before calling `NewAPIClient`.

```go
import (
    "fmt"
    "github.com/okta/okta-sdk-golang/v6/okta"
)

func main() {
    cfg, err := okta.NewConfiguration(
        okta.WithOrgUrl("https://{yourOktaDomain}"),
        okta.WithToken("{apiToken}"),
    )
    if err != nil {
        fmt.Printf("configuration error: %v\n", err)
        return
    }
    client := okta.NewAPIClient(cfg)
    _ = client
}
```

## API service properties

Every property on `APIClient` is an API service interface. Call the service methods directly on the returned client — there is no separate service lookup step.

```go
user, resp, err := client.UserAPI.GetUser(ctx, "{userId}").Execute()
```

The services are grouped below by functional area.
